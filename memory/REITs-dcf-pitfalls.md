# 酒店类REIT DCF建模常见陷阱与核对清单

## 陷阱1: GOP数据来源错误

| 项目 | 说明 |
|------|------|
| **问题** | 利润表GOP vs 管理报表GOP口径不同 |
| **核对** | 明确提取的是哪个报表的数据 |
| **修正** | 管理报表更贴近实际经营，优先使用 |

**如何区分：**
- 管理报表GOP：通常包含更完整的经营数据
- 利润表GOP：会计口径，可能有调整项

---

## 陷阱2: 评估基准日混淆

| 项目 | 说明 |
|------|------|
| **问题** | 混淆2025年数据 vs 2026年数据 |
| **核对** | 明确评估基准日（如2025-12-31） |
| **修正** | 基准日之后的第一年是DCF首年（如2026年） |

**关键检查点：**
```
评估基准日: 2025-12-31
DCF首年: 2026年 (t=1)
土地使用权剩余年限: 从2025-12-31起算
```

---

## 陷阱3: 增长率假设单一化

| 项目 | 说明 |
|------|------|
| **问题** | 不同项目使用相同增长率 |
| **核对** | 检查招募说明书是否分项目披露增长率 |
| **修正** | 按项目分别设置增长率 |

**示例（不同项目不同增长率）：**
```python
growth_rates = {
    "project_a": {"y2": 0.02, "y3": 0.03, "y4_10": 0.025, "y11+": 0.02},
    "project_b": {"y2": 0.01, "y3": 0.02, "y4_10": 0.025, "y11+": 0.02}
}
```

---

## 陷阱4: 折旧重复扣除

| 项目 | 说明 |
|------|------|
| **问题** | GOP已扣除折旧，NOI计算又扣一次 |
| **核对** | GOP是否包含折旧？（通常包含） |
| **修正** | GOP - 折旧 = 经营现金流，不要再扣折旧 |

**公式确认：**
```
GOP = 营业收入 - 营业成本 - 税金及附加 - 管理费用 - 折旧
NOI = GOP + 商业净收入 - 其他费用 - Capex  (不再扣折旧)
```

---

## 陷阱5: 对比基准错误

| 项目 | 说明 |
|------|------|
| **问题** | 与募集资金对比（包含资本结构） |
| **核对** | 明确对比目标是资产评估值 |
| **修正** | 始终与资产评估值对比 |

**区分：**
- 资产评估值：纯资产价值（DCF应该接近这个）
- 募集资金：包含发行费用、资本结构安排等

---

## 陷阱6: 终值处理分歧

| 项目 | 说明 |
|------|------|
| **问题** | 使用永续增长模型 vs 持有到期模型 |
| **核对** | 土地使用权到期后是否有残值？ |
| **建议** | 保守起见使用持有到期（残值=0） |

**推荐做法：**
- 酒店REIT通常采用**持有到期模型**
- 土地使用权到期后残值 = 0
- 不进行永续增长假设

---

## 陷阱7: 含税/不含税口径混淆

| 项目 | 说明 |
|------|------|
| **问题** | ADR含税但收入不含税，导致计算差异 |
| **核对** | 检查ADR和收入的增值税口径是否一致 |
| **修正** | 明确标注ADR含税率，收入需转换为含税价或统一口径 |

**关键公式：**
```
# 增值税计算（酒店收入税率6%）
不含税收入 = 含税收入 / (1 + 6%)
增值税额 = 含税收入 × 6% / (1 + 6%)

# ADR与客房收入口径统一
ADR(含税) × 房间数 × OCC × 365 = 含税客房收入
含税客房收入 / 1.06 = 不含税客房收入（招募书披露值）
```

**避坑要点：**
- ADR通常报价为含税价（含6%增值税）
- 招募说明书披露的客房收入通常为不含税价
- 计算差异约5.66%（即6%/1.06），需在参数提取时标注口径

**示例：**
| 指标 | 含税口径 | 不含税口径 | 差异 |
|------|----------|------------|------|
| ADR | 495.58元 | 467.53元 | -5.66% |
| 客房收入 | 12,735万 | 12,015万 | -5.66% |

## 陷阱8: 增值税重复扣除

| 项目 | 说明 |
|------|------|
| **问题** | 推导NOI时单独计算并扣除增值税，但营业收入已是不含税净额 |
| **核对** | 中国GAAP"营业收入"是否含增值税？（不含） |
| **修正** | 禁止在NOI推导中单独扣除增值税 |

**关键规则：**
```
中国GAAP: 营业收入 = 不含增值税净额
税金及附加 ≠ 增值税（仅含城建税、教育费附加等）
NOI推导中禁止再单独计算和扣除增值税
违反后果: NOI偏低800-1000万（约10%）
```

---

## 陷阱9: 成本口径混用（REITs明细 vs 历史利润表）

| 项目 | 说明 |
|------|------|
| **问题** | 同时使用历史利润表的"营业成本"和独立的物业费/保险，导致双重计算 |
| **核对** | 成本是用历史利润表口径还是REITs明细口径？ |
| **修正** | 二选一，REITs建模优先用明细口径 |

**两种口径：**
```
口径A: 历史利润表"运营成本(不含折旧)" = 已含物业/保险
口径B: REITs明细 = 运营明细 + 物业(独立合同) + 保险(独立合同)
两者不可混用
```

---

## 陷阱10: 管理费用定义错误

| 项目 | 说明 |
|------|------|
| **问题** | 使用利润表"管理费用"（含公司行政开支），而非酒店运营管理费 |
| **核对** | 管理费是GOP×费率还是利润表科目？ |
| **修正** | 管理费 = GOP × fee_rate（通常3%），付给酒店管理公司 |

**区分：**
```
利润表"管理费用" = 公司行政 + 办公租金 + 法务审计 + ...
                   → REITs后由基金管理人承担，不计入项目NOI
酒店运营管理费 = GOP × 3%
                   → 付给酒店管理公司（如华住），应从NOI扣除
```

---

## 陷阱11: NOI验证失败后回退到招募值

| 项目 | 说明 |
|------|------|
| **问题** | 推导NOI与招募NOI/CF差异>5%时回退使用招募值 |
| **核对** | 推导NOI和招募NOI的口径是否相同？ |
| **修正** | 始终使用推导NOI，差异来自口径不同而非计算错误 |

**正确处理：**
```
推导NOI ←→ 招募NOI/CF 差异:
  ≤5%: 标注 [PASS]
  >5%: 标注 [差异X.X%]，仍使用推导值
始终不回退到招募值
```

---

## 陷阱12: 税金及附加用推导值而非实际值

| 项目 | 说明 |
|------|------|
| **问题** | 从房产原值推导税金远高于实际缴纳值（差8%-33%） |
| **核对** | 税金来源是推导还是实际？ |
| **修正** | 优先使用实际缴纳值，推导值仅对照参考 |

**差异原因：**
- 税收减免/优惠政策
- 基数确认差异（账面vs评估vs原始成本）
- 城建税/教育费附加计算基数不同

## 建模前核对清单

### 数据提取阶段
- [ ] 确认招募说明书版本（草稿/终稿）
- [ ] 标记所有参数来源页码
- [ ] 区分管理报表 vs 利润表数据
- [ ] 确认评估基准日
- [ ] **标注含税/不含税口径**（ADR含税6%，收入需转换为含税价对比）

### 参数确认阶段
- [ ] 各项目土地使用权剩余年限
- [ ] 各项目分段增长率
- [ ] 折现率/报酬率
- [ ] 首年及预测期Capex

### 建模阶段
- [ ] GOP数据来源标注清楚
- [ ] 折旧是否已剔除确认
- [ ] 房产税/土地税处理确认（用实际缴纳值）
- [ ] 分段增长率实现正确
- [ ] **增值税未被重复扣除**
- [ ] **成本口径统一（REITs明细 or 历史利润表，不可混用）**
- [ ] **管理费 = GOP×fee_rate（非利润表"管理费用"）**
- [ ] **NOI使用推导值（不回退到招募值）**
- [ ] **部分年限已正确处理**

### 验证阶段
- [ ] 与资产评估值对比
- [ ] 单房估值合理性检查
- [ ] 隐含Cap Rate合理性
- [ ] 敏感性分析覆盖关键参数

## 模型审计报告必备内容

1. **Input层级图**: 明确哪些是直接摘录，哪些是计算得出
2. **参数来源表**: 每个参数的来源页码
3. **假设分类表**: 哪些是招募书给定，哪些是用户假设
4. **计算步骤详解**: 公式、示例、逐年计算
5. **对比分析**: DCF估值 vs 资产评估值
6. **敏感性分析**: 折现率和增长率敏感性
7. **风险提示**: 关键假设风险

## 快速问题诊断

| 症状 | 可能原因 | 检查点 |
|------|----------|--------|
| DCF估值远高于评估值 | 折旧重复扣除 / Capex未扣除 | 检查NOI计算公式 |
| DCF估值远低于评估值 | 增长率设置过低 / 折现率过高 | 核对增长率参数 |
| DCF估值低10%左右 | 增值税重复扣除 / 成本口径混用 / 管理费定义错 | 逐项检查IR-1~IR-3 |
| 单房估值异常 | 房间数错误 / 总估值计算错误 | 检查房间数单位 |
| 隐含Cap Rate异常 | NOI口径错误 | 确认NOI计算逻辑 |
| **收入计算差异3-6%** | **ADR含税 vs 收入不含税** | **检查增值税口径统一性（6%）** |
| **税金偏高8-30%** | **推导值vs实际值差异** | **用实际缴纳值，推导值仅对照** |
| **NOI差异>5%** | **成本口径(REITs明细vs利润表)不同** | **仍用推导值+标注差异** |
