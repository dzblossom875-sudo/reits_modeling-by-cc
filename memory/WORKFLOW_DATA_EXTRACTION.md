# 数据提取工作流规范

## 核心原则

> **所有计算涉及的数据必须在第一步（参数提取阶段）完成提取、验证并存档。**
>
> **禁止行为**: 在建模/比对阶段发现数据缺失而回头补充PDF提取。

---

## 三阶段工作流

### 阶段一：参数提取（Parameter Extraction）

**目标**: 从招募说明书完整提取所有DCF建模所需参数

**输入**: 招募说明书PDF
**输出**: `data/huazhu/extracted_params_detailed.json`

#### 检查清单

- [ ] **项目基础信息** (Tier 1)
  - [ ] 项目名称、品牌、位置
  - [ ] 房间数（分品牌明细）
  - [ ] 建筑面积、土地面积
  - [ ] 土地使用权剩余年限
  - [ ] 开业时间

- [ ] **运营指标** (Tier 1-2)
  - [ ] ADR（分品牌）
  - [ ] OCC（分品牌）
  - [ ] RevPAR（如有）
  - [ ] **验证**: ADR × OCC = RevPAR ?

- [ ] **收入数据** (Tier 2)
  - [ ] 客房收入（招募说明书财务报表）
  - [ ] **验证**: ADR × 房间数 × OCC × 365 ≈ 客房收入 ?
  - [ ] OTA收入占比
  - [ ] 餐饮收入及占客房收入比例
  - [ ] 其他收入及占客房收入比例
  - [ ] 商业租金收入
  - [ ] 商业物业费收入

- [ ] **成本费用** (Tier 2)
  - [ ] 运营费用明细（人工、餐饮、清洁、能源、维护、营销、数据系统、其他）
  - [ ] 物业费用（独立合同：单价 × 面积 × 12）
  - [ ] 保险费用（独立合同）
  - [ ] 房产税（酒店从价、商业从租）— **提取实际缴纳值**
  - [ ] 土地使用税 — **提取实际缴纳值**
  - [ ] 管理费率（GOP×fee_rate，通常3%）
  - [ ] **注意**: 识别成本中的折旧金额（需剔除）
  - [ ] **注意**: 标注口径（REITs明细 vs 历史利润表），不可混用

- [ ] **资本性支出** (Tier 2)
  - [ ] 年度Capex预测（至少前3年）
  - [ ] 翻新周期
  - [ ] 大额翻新成本

- [ ] **估值参数** (Tier 3)
  - [ ] 折现率/报酬率
  - [ ] 分段增长率（第2年、第3年、第4-10年、第11年起）
  - [ ] CPI基准增长率

- [ ] **来源标注**
  - [ ] 每个参数标注来源页码
  - [ ] 区分"招募书给定"vs"估算参数"

#### 交叉验证规则

| 验证项 | 验证方法 | 可接受差异 |
|--------|---------|-----------|
| 客房收入 | ADR公式计算 vs 招募财务报表 | ±5% |
| 总房间数 | 分品牌房间数之和 vs 总数 | 0 |
| RevPAR | ADR × OCC vs 招募RevPAR | ±1% |
| 收入比例 | 餐饮/其他占比 vs 历史平均 | ±2% |

---

### 阶段二：建模（Modeling）

**目标**: 基于已存档参数进行计算，**不再新增参数提取**

**输入**: `extracted_params_detailed.json`
**输出**: DCF模型结果

#### 禁止行为

❌ **禁止** 在建模阶段发现数据缺失而临时读取PDF
❌ **禁止** 在建模阶段修改 `extracted_params_detailed.json`
❌ **禁止** 使用未在第一步验证的参数

#### 允许行为

✅ **允许** 发现第一阶段数据错误时，回到阶段一修正后重新提取
✅ **允许** 使用第一阶段已标记的"估算参数"
✅ **允许** 基于第一阶段参数进行敏感性分析

---

### 阶段三：比对（Comparison）

**目标**: 计算结果与招募说明书进行逐项比对，识别差异

**输入**: DCF计算结果 + 招募说明书关键指标
**输出**: 差异分析报告

#### 差异处理流程

```
发现差异
    ↓
分析差异原因
    ↓
├─ 数据提取错误？ → 回到阶段一修正 → 重新建模 → 重新比对
├─ 计算逻辑错误？ → 修正模型 → 重新计算 → 重新比对
└─ 口径差异（如含税vs不含税）？ → 记录在案 → 报告说明
```

## 本次实践案例分析

### 错误案例1: 广州项目客房收入数字错位

| 问题 | 原因 | 应在阶段一完成的验证 |
|------|------|---------------------|
| 12,015.67万误写为13,215.67万 | 数字提取时位序错误 | ADR公式计算 vs 招募值交叉验证 |

### 错误案例2: ADR含税口径未明确

| 问题 | 原因 | 应在阶段一完成的标注 |
|------|------|---------------------|
| 计算值与招募值差异3-5% | ADR含税，收入不含税 | 在json中标注"ADR含税6%" |

### 错误案例3: 分品牌数据缺失

| 问题 | 原因 | 应在阶段一完成的提取 |
|------|------|---------------------|
| 广州项目使用统一ADR/OCC | 美居/全季分别的ADR/OCC未提取 | 从Page 161提取分品牌数据 |

## 质量门禁

进入下一阶段前必须满足：

| 阶段 | 门禁条件 |
|------|---------|
| 阶段一 → 阶段二 | `extracted_params_detailed.json` 存在且通过交叉验证 |
| 阶段二 → 阶段三 | 所有计算基于json文件，无新增PDF读取 |
| 阶段三 → 完成 | 差异分析报告已归档 |
