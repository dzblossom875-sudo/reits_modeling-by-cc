# NOI推导与DCF建模调试记录

本文档记录酒店REITs DCF建模过程中的所有调试、修正和关键结论，按发现顺序排列。

---

## 1. 增值税重复扣除

**问题**: 从收支明细推导NOI时，单独计算了增值税（酒店6%+商业9%）并从收入中扣除，导致NOI偏低约800-1000万。

**根因**: 中国GAAP下，利润表"营业收入"已是**不含增值税的净额**。"税金及附加"科目也不含增值税本身（仅含城建税、教育费附加等附加税）。在NOI推导中再单独扣除增值税，属于重复扣除。

**修正**: NOI推导中不再单独计算和扣除增值税。

**规则**: 
```
中国GAAP营业收入 = 不含增值税净额
税金及附加 ≠ 增值税，仅含附加税种
NOI推导中禁止单独扣除增值税
```

---

## 2. ADR含税与收入不含税的口径差异

**问题**: ADR（日均房价）为含税报价（宾客支付价），first_year_amount为不含税收入。两项目隐含税率不一致（广州3.3% vs 上海4.8%），实际应该相同。

**根因**: ADR是宾客端含税报价，first_year_amount是利润表不含税收入，两者差异来自增值税。两项目看似不同税率是因为分别反推，实际应统一用6%（酒店住宿业一般纳税人标准税率）。

**修正**: 统一用6%增值税率做价税分离。ADR含税计算值÷1.06=不含税值，与报表不含税收入对比。

**验证结果**:
| 项目 | ADR含税值 | ÷1.06 | 报表不含税 | 差异 |
|------|----------|-------|----------|------|
| 广州 | 12,408万 | 11,705万 | 12,016万 | +2.7% |
| 上海 | 3,179万 | 3,000万 | 3,035万 | +1.2% |

剩余2-3%差异来自ADR加权方式/季节性/房型差异，属正常范围。

**规则**:
```
ADR为含税报价 → 统一6%增值税率
对比口径必须一致: 含税vs含税 或 不含税vs不含税
禁止用不同隐含税率反推
```

## 3. 物业费和保险双重计算

**问题**: 模型将物业费（广州436万/上海144万）和保险（广州45万/上海18万）单独列支，但这些费用已包含在历史利润表的"营业成本(不含折旧)"中，导致成本双重计算。

**根因**: 历史利润表的"营业成本"是综合科目，已含物业管理费和保险。但REITs后成本重分类，物业和保险变成独立合同。如果同时用历史利润表的营业成本基数又加上独立的物业/保险，就会重复。

**修正**: REITs后应使用明细成本表（运营明细+物业+保险），不再使用历史利润表的"运营成本(不含折旧)"。两种口径不能混用。

**规则**:
```
二选一，不可混用:
  口径A: 历史利润表"运营成本(不含折旧)" = 已含物业/保险
  口径B: REITs明细 = 运营明细 + 物业(独立) + 保险(独立)
REITs建模应使用口径B（反映发行后的成本结构）
```

## 4. 管理费用口径错误

**问题**: 使用利润表"管理费用"（广州449万/上海160万）作为NOI扣减项，导致NOI偏低。上海管理费用标注为"估算值"，数据不可靠。

**根因**: 利润表"管理费用"包含**公司层面行政开支**（办公租金、法务、审计费等），REITs后这些费用由基金管理人承担，不再计入项目NOI。项目NOI只扣减付给酒店管理公司（华住）的**运营管理费**。

**修正**: 管理费 = GOP × 管理费率（3%）。来源：招募说明书明确定义"付给酒店管理公司的费用，按GOP×基准费率缴纳"。

**验证**: 
- 广州: 隐含管理费=397万(4.6%GOP), 历史管理费用=449万。差额52万为公司行政开支。
- 上海: 隐含管理费=14万(0.8%GOP), 历史管理费用=160万。差额146万为公司行政开支。

**规则**:
```
管理费 = GOP × fee_rate（通常3%）
NOT 利润表"管理费用"（含公司行政，REITs后不适用）
```

## 5. 房产税推导值与实际缴纳值差异

**问题**: 从房产原值推导的税金（广州930万/上海248万）远高于实际缴纳值（广州856万/上海186万），差异达8.6%-33.5%。

**根因**: 推导使用的是账面房产原值×(1-30%)×1.2%，但实际可能存在：
- 税收优惠/减免政策
- 从价计征的基数确认差异（账面值vs评估值vs原始成本）
- 城建税/教育费附加的计算基数差异

**修正**: 税金及附加使用**实际缴纳值**（招募说明书历史数据），推导值仅作参考对照记录。

**规则**:
```
税金及附加 = 实际缴纳值（优先）
推导值仅作对照（标注差异百分比和可能原因）
```

## 6. NOI验证失败时的处理策略

**问题**: 初版逻辑在推导NOI与招募NOI/CF差异>5%时，回退使用招募值。但用户指出应始终使用推导值。

**根因**: 推导NOI反映REITs后的实际成本结构（成本重分类），与招募的利润表口径本就不同。差异是口径差异，不是推导错误。

**修正**: 始终使用推导NOI作为DCF输入。差异≤5%标注PASS，>5%标注具体差异率但仍然使用。

**规则**:
```
NOI始终使用推导值
差异≤5%: 标注 [PASS]
差异>5%: 标注 [差异X.X%]，仍使用推导值
不回退到招募值
```

## 7. 估值差异分析（DCF vs 招募资产评估值）

**问题**: DCF模型估值14.03亿 vs 招募资产评估值15.91亿，差异-11.8%。

**分析**:

| 因素 | 影响方向 | 估计影响 |
|------|---------|---------|
| 运营成本口径（REITs明细>历史利润表） | 低估NOI | -3~5% |
| 折现时点约定（期末vs期中） | 低估约 | -3% |
| GOP率固定假设（实际可能改善） | 低估 | -2~3% |
| 无残值（用户确认） | 合理 | 0 |
| 折现率5.75%（用户确认不变） | 合理 | 0 |

**结论**: 差异主要来自成本口径差异和折现约定，属模型方法论差异，非计算错误。

## 8. 部分年限处理

**问题**: 广州剩余19.28年、上海30.65年，int()取整导致丢失0.28年/0.65年的现金流。

**修正**: 增加部分年处理逻辑：完整年+部分年(按比例)，折现因子为(1+r)^(full_years+partial)。

---

## 9. GOP率波动与稳定性

**发现**: 历史GOP率波动较大：
- 广州: 60.1%→65.6%（3年波动5.5个百分点）
- 上海: 51.1%→54.5%（3年波动3.4个百分点）

**影响**: 模型假设所有收入/成本同比例增长（GOP率固定）。实际上固定成本（物业费/保险）不随收入增长，GOP率应逐年改善。这是模型低估的原因之一。

---

## 10. 上海运营成本明细偏高

**问题**: 上海REITs明细运营成本(1,436万)比历史(1,323万)高113万(+8.5%)，导致推导NOI与招募NOI/CF差异-8.6%。

**分析**:
- 运营明细项合计(1,274万)已高于historical_avg_ratio×revenue(35%×3,314=1,160万)
- 加上独立物业费(144万)和保险(18万)后更高
- 可能原因: 明细估算值偏高（特别是marketing 180万、maintenance 140万）

**处理**: 使用明细值+标注差异。差异超过5%时仍使用推导值。

## 11. 商业购物中心NOI — 增值税处理（商业REIT通用）

**问题**: 商业REIT mall模型中，收入已按不含税口径列示，但仍在税金中单独扣除增值税（约6,000万+/年），导致模型估值偏低约14%。

**根因**: 增值税为价外税（价税分离机制）：
- 租户支付含税租金，REIT收取后将VAT部分汇缴税局
- 财务报告中"收入"已是税后净额（不含VAT）
- VAT对REIT持有人是纯过路资金，无实际成本
- 在不含税收入基础上再扣VAT = 双重扣除

**修正**: 增值税不计入成本。仅附加税（城建税+教育费附加 = VAT × 12%）为真实税负，计入税金。
- 修正前Y1 FCF ≈ 57,419万，估值 ≈ 786,384万（vs报告920,500万，差-14.6%）
- 修正后Y1 FCF ≈ 63,446万，估值 ≈ 868,036万（差-5.7%）

**规则**:
```
商业REIT（含增值税区分）:
  收入口径 = 不含税净额（固定租金/物管费/停车等均为不含税）
  增值税 = 纯过路，不进NOI计算
  附加税 = VAT × 12% → 真实税负，计入tax
  房产税（从租）= 12% × 不含税租金（固定+停车）
```

**注意**: 需单独保留VAT计算逻辑用于附加税基数（`vat_total`字段保留，仅不加入`total_tax`）。

## 12. 商业购物中心NOI — 物业管理费成本口径

**确认**: 物业管理费支出（成本端）= 50% × 物管收入（含税口径）
- 物管收入不含税14,134万 × (1+6%) = 14,982万（含税）
- 成本 = 14,982 × 50% = 7,491万
- **用户明确确认：50%基准是含税口径**，非不含税

## 13. 商业购物中心 — 参数提取关键陷阱

**陷阱清单**（基于成都万象城项目实战）:

| 字段 | 常见误区 | 正确做法 |
|------|---------|---------|
| 固定租金Y1 | 混淆含税/不含税 | 由报告披露的Capex绝对值反推：总收入=Capex/2.5% |
| 停车场收入 | 整租合同是含税金额 | 先记含税，再÷(1+9%)转不含税 |
| 提成租金占比 | 理解为"提成/固定" | 正确：提成/(固定+提成)=22%，公式：提成=22/78×固定 |
| 增值税率 | 混用5%（不含税）和4.76%（含税净率） | 明确区分：4.76% = 5%/(1+5%)；仅用于附加税基数 |
| 一二期增值税 | 忘记一期简易计税(5%)二期一般计税(9%) | 需按一二期面积/收入比例分别计算 |
| 收缴率 | 忽略Y6翻新时出租率降至90% | 读取报告逐年收缴率计划表（Page 76附近） |
| 增长率 | 主次力店/专门店/超市影院不同 | 读报告专用表格（Page 81右下角） |

## 核心公式（最终版）

```
营业收入(不含税) = first_year_amount（ADR含6%增值税，已价税分离）
运营成本(REITs口径) = 运营明细 + 物业费(独立) + 保险(独立)
GOP = 营业收入 - 运营成本 - 税金及附加(实际缴纳)
管理费 = GOP × 3%（酒店运营管理费，付给华住）
NOI/CF = GOP - 管理费 - Capex
DCF = Σ(NOI×累积增长 - Capex) / (1+r)^t  （含部分年，无残值）
```
